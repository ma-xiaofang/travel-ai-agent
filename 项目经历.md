项目名称： 旅途 · AI 旅行规划助手 
技术栈： NestJS 11 · TypeScript · LangGraph 1.2.x · LangChain.js 1.x · DeepSeek Chat · PostgreSQL · pgvector · Prisma · Vue 3 · Element Plus · Pinia · SSE · Tavily · RAG
项目描述： 面向个人旅行者的 AI 规划助手，用户输入目的地和需求，系统按"RAG 知识库 → Tavily 联网搜索 → 内置参考数据"三级回退策略自主调度 10 个工具，完成天气查询、景点推荐、行程生成、预算估算、签证信息、货币兑换、打包清单、旅行短语翻译、联网搜索、会话标题生成共 10 项旅行任务，并以 SSE 流式输出结构化旅行方案。配套 Vue 3 + Element Plus 管理后台，提供仪表盘、知识库运营、会话观测、用户管理等运营能力。
工作内容：
1. 基于 LangGraph StateGraph 构建 ReAct 模式 Agent，实现"LLM 推理 → 工具调用 → 结果回传 → 继续推理"的多步自主循环，支持 recursionLimit 防无限迭代。
2. 基于 LangChain.js Tool 机制，完成 10 类工具注册、调用协议定义与路由接入，其中 8 个旅行工具采用"RAG → Tavily → mock"三级数据回退策略。
3. 使用 Zod 参数校验框架，为全部工具编写入参与出参约束规则，并落地统一 JSON 返回结构。
4. 基于 NestJS 模块化架构，开发 SSE 流式 + REST 同步双通道 Agent 对话接口，并实现会话创建、归属校验、历史查询等会话管理 API。
5. 实现 DeepSeek 思维链（reasoning_content）的流式提取与 SSE 推送，前端可折叠展示"思考过程"面板。
6. 基于 Prisma ORM，完成 ChatSession、ChatMessage、KnowledgeCollection、KnowledgeDocument、KnowledgeChunk 等核心数据模型设计与数据库访问层封装。
7. 基于 PostgreSQL + pgvector 搭建向量存储能力，实现文档分块、向量化入库与余弦相似度 Top-K 检索，并编写完整数据迁移脚本。
8. 使用 mxbai-embed-large（1024 维）嵌入模型 + RecursiveCharacterTextSplitter（chunkSize=500, chunkOverlap=50）对知识文档进行分块与向量化存储，构建 RAG 知识库基础设施。
9. 基于 Chroma + PGVector 双向量存储适配层（VectorStoreService 接口），实现 addDocuments、similaritySearch、deleteByDocId 等统一抽象，支持存储后端灵活切换。
10. 集成 Tavily 联网搜索引擎，实现 searchRaw / searchForTool / search 三层调用封装，支持 topic、timeRange、searchDepth 参数化配置，并在未配置 API Key 时优雅降级。
11. 使用 Prompt 模板工程方法，编写系统提示词、工具描述词、RAG 问答提示词与多轮对话约束模板，并针对 DeepSeek 模型特性优化指令表述。
12. 基于 NestJS 守卫 + JWT 策略，实现 ADMIN / USER 角色权限分离，管理后台 API 仅允许 ADMIN 角色访问。
13. 开发管理后台仪表盘统计服务，实时聚合知识文档数量、索引状态、会话总量、消息总量等运营指标。
14. 开发知识库管理服务（集合 CRUD + 文档 CRUD + 分块查询 + 单文档索引触发），支持文档的草稿、处理中、已索引、失败全生命周期管理。
15. 开发会话观测服务（会话列表/详情/删除 + 消息历史查看），支持按 userId、关键词、时间范围筛选与分页查询。
16. 开发用户管理服务（用户列表/详情/角色变更），支持按邮箱、角色、状态筛选与分页查询。
17. 基于样例回放评测方式，构建"行程/预算/签证/结构化输出"测试用例并持续迭代。
18. 使用异常处理中间层机制，编排超时、限流、工具失败等场景的降级与重试流程。
19. 基于日志（NestJS Logger）与配置中心（ConfigService + dotenvx），完善运行日志、环境变量管理与服务健康检查能力。
20. 基于 @langchain/openai 补丁机制（pnpm patches），修复 convertMessagesToCompletionsMessageParams 未透传 reasoning_content 导致的工具调用链第二轮 400 错误。
21. 使用 Vue 3 Composition API + Element Plus 组件库，开发管理后台 14 个页面，涵盖仪表盘、知识库（集合/文档/分块/RAG 调试）、会话观测（会话管理/消息管理/对话）、用户管理、工具详情、系统状态、登录页。
22. 基于 Pinia Store 分层设计，封装消息写入、流式追加、会话重置等核心状态操作方法。
23. 基于 Fetch + ReadableStream 流读取能力，实现前端对 SSE 增量内容的分片消费与拼接展示，支持 text / reasoning / session 三类事件的差异化处理。
24. 使用 markdown-it + markdown-it-multimd-table + DOMPurify + highlight.js 渲染消息内容，实现 Markdown 解析、复杂表格支持、代码高亮与 XSS 安全过滤。
25. 基于 Hash 路由 + 嵌套路由 + 侧栏布局，开发管理后台路由体系与面包屑导航。
26. 基于 Axios 请求封装机制，统一前端 API 调用拦截器（自动解包 res.data）、超时配置与错误处理策略。
27. 基于 CORS 与 Vite 代理联调配置，打通管理后台到 NestJS 后端的本地开发转发链路。
28. 基于 Prisma Client 查询能力，开发按 sessionId 加载历史消息（最近 20 条）并按时间排序注入上下文的会话记忆持久化流程，超出上限自动清理旧消息。
29. 基于 Node.js 环境变量管理方式（dotenvx），编写 LLM、数据库、向量存储、Tavily API、JWT 等分环境配置模板。
30. 基于接口联调清单与回归用例，组织前后端联调、发布验收与版本回归测试流程。
31. 基于项目文档体系建设，编写架构说明、工具接入规范、LangGraph 流式对话流程说明与常见故障排查手册。

项目亮点：
1. 构建多工具协同的 AI 旅行规划链路，覆盖"信息查询 → 行程规划 → 决策支持 → 会话管理"完整闭环。
2. 创新性地设计"RAG 知识库 → Tavily 联网搜索 → 内置参考数据"三级数据回退策略，兼顾数据准确性、实时性与服务可用性。
3. 采用 SSE 流式输出 + DeepSeek 思维链可视化，复杂任务下可边推理边展示，降低用户等待焦虑。
4. 架构具备良好扩展性，新增工具通过标准 Tool 接口注册即可接入；向量存储通过 VectorStoreService 接口抽象，支持 PGVector / Chroma 灵活切换。
5. 配套完整的管理后台，提供知识库运营（文档入库→分块→索引全流程）、会话观测（对话历史回放）、用户管理等运营能力。

技术难点与解决方案：
1. 难点：多工具并行语义下容易出现参数不完整、工具误选和结果格式不统一。
   解决：通过工具描述词优化 + Zod 强校验 + 统一 JSON 响应协议，配合三级数据回退与失败兜底回复，提升调用成功率与结果可解析性。
2. 难点：长文本生成场景下用户等待时间长、交互割裂。
   解决：引入 SSE 增量推送 + reasoning_content 流式输出 + 前端分片渲染策略，优先返回关键结论再补全细节，优化感知性能。
3. 难点：DeepSeek 模型工具调用链中 reasoning_content 字段丢失导致第二轮请求 400 错误。
   解决：通过 pnpm patches 机制修复 @langchain/openai 的消息序列化逻辑，在 completionParam 中透传 reasoning_content，确保多轮工具调用链路通畅。
4. 难点：向量存储与关系型数据的一致性问题（文档状态、分块元数据、embeddingId 关联）。
   解决：采用"先写 Prisma 状态 → 向量入库 → 回写 embeddingId → 更新状态"的事务化流程，异常时自动标记 FAILED 并记录错误信息。
