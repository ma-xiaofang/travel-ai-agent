项目名称： 旅途 · AI 旅行规划助手 
技术栈： NestJS · TypeScript · LangGraph 1.2.x · LangChain.js 1.x · GPT-4o · DeepSeek Chat · PostgreSQL · pgvector · Prisma · Vue3 · SSE
项目描述： 面向个人旅行者的 AI 规划助手，用户输入目的地和需求，系统自动调度多个工具完成天气查询、行程生成、预算估算、签证信息、货币兑换、景点推荐、打包清单、旅行短语翻译共 8 项旅行任务，最终以流式方式输出结构化旅行方案。
工作内容：
1. 基于 LangGraph StateGraph 构建 ReAct 模式 Agent，实现工具调用的自主路由和多步推理。  
2. 基于 LangChain.js Tool 机制，完成 8 类旅行工具注册、调用协议定义与路由接入。  
3. 使用 Zod 参数校验框架，编写工具入参与出参约束规则并落地统一返回结构。  
4. 基于 NestJS 模块化架构，开发 SSE + REST 双通道接口并实现会话管理相关 API。  
5. 使用 TypeScript 类型系统，重构请求与响应 DTO，统一前后端字段定义与错误码语义。  
6. 基于多模型配置层设计，接入 Ollama、DeepSeek、GPT 系列模型并实现按场景切换。  
7. 使用 Vue3 + Pinia 状态管理方案，开发聊天消息流式渲染、加载状态与错误提示逻辑。  
8. 基于 Fetch + ReadableStream 流读取能力，实现前端对 SSE 增量内容的分片消费与拼接展示。  
9. 使用 Prisma ORM，完成会话、消息等核心数据模型设计与数据库访问层封装。  
10. 基于 PostgreSQL + pgvector 能力，搭建向量字段与检索预留结构并编写数据迁移脚本。  
11. 使用 Prompt 模板工程方法，编写系统提示词、工具描述词与多轮对话约束模板。  
12. 基于样例回放评测方式，构建“行程/预算/签证/结构化输出”测试用例并持续迭代。  
13. 使用异常处理中间层机制，编排超时、限流、工具失败等场景的降级与重试流程。  
14. 基于日志与配置中心方案，完善运行日志、环境变量管理与服务健康检查能力。  
15. 使用接口联调清单与回归用例，组织前后端联调、发布验收与版本回归测试流程。  
16. 基于项目文档体系建设，编写架构说明、工具接入规范与常见故障排查手册。  
17. 基于 OpenWeatherMap API 接入方案，开发天气工具的真实数据查询与模拟数据双通道逻辑。  
18. 使用 Axios 请求封装机制，统一前端 API 调用拦截器、超时配置与错误处理策略。  
19. 基于 Hash 路由与页面模块拆分，开发聊天主页与历史记录页并实现路由级状态切换。  
20. 使用 Pinia Store 分层设计，封装消息写入、流式追加、会话重置等核心状态操作方法。  
21. 基于 CORS 与代理联调配置，打通 Vite 到 NestJS 的本地开发转发链路。  
22. 使用 Node.js 环境变量管理方式，编写 LLM、数据库、第三方 API 的分环境配置模板。  
23. 基于用户输入预处理规则，开发目的地、预算区间、出行天数等字段的格式化解析逻辑。  
24. 使用工具输出标准化策略，统一天气、签证、汇率等异构数据的字段命名与展示顺序。  
25. 基于错误码分层设计，定义模型层、工具层、接口层异常分类并补充对应处理分支。  
26. 使用服务健康检查与工具可用性探测机制，开发 Agent 启动前依赖自检流程。  
27. 基于 Jest 单元测试框架，编写工具参数校验与核心服务方法的基础测试用例。  
28. 使用接口示例与时序图文档，沉淀“用户输入 → Agent 决策 → 工具调用 → 流式返回”调用链说明。  
29. 基于 mxbai-embed-large（1024 维）嵌入模型，对热门城市景点信息进行向量化存储并使用余弦相似度检索 Top-K 结果后拼装检索上下文。  
30. 基于 Prisma Client 查询能力，开发按 userId 加载历史消息并按时间排序注入上下文的会话记忆持久化流程。  
31. 使用 markdown-it + markdown-it-multimd-table + DOMPurify 渲染消息内容，实现 Markdown 解析、复杂表格支持与 XSS 安全过滤。  

项目亮点：
1. 构建多工具协同的 AI 旅行规划链路，覆盖“信息查询 → 行程规划 → 决策支持”完整闭环。  
2. 采用流式输出显著优化用户感知，复杂任务下可实现边推理边展示，降低等待焦虑。  
3. 架构具备良好扩展性，新增工具可通过标准协议快速接入，支持后续接入真实第三方 API 与企业级能力。  

技术难点与解决方案：
1. 难点：多工具并行语义下容易出现参数不完整、工具误选和结果格式不统一。  
   解决：通过工具描述词优化 + Zod 强校验 + 统一响应协议，配合失败重试与兜底回复，提升调用成功率与结果可解析性。  
2. 难点：长文本生成场景下用户等待时间长、交互割裂。  
   解决：引入 SSE 增量推送与前端分片渲染策略，优先返回关键结论，再补全细节，优化感知性能。  
3. 难点：本地模型与云端模型能力差异导致输出稳定性波动。  
   解决：抽象统一模型配置层，按场景切换模型与温度参数，并通过回放用例进行质量对齐。  

项目成果：
1. 完成从需求输入到结构化旅行方案输出的全流程闭环，支持行程、预算、签证、天气等 8 类核心能力。  
2. 建立可扩展工具框架，新增旅行工具时仅需实现标准接口并注册即可生效，显著降低功能迭代成本。  
3. 沉淀可复用的 Agent 工程化模板，可迁移到“本地生活推荐、攻略问答、智能客服”等多工具编排场景。  

个人成长：
1. 深入掌握 LangGraph 在生产化场景中的状态编排、工具路由与异常治理实践。  
2. 提升了从 LLM 应用设计到前后端联调、数据建模、交互优化的一体化交付能力。  
3. 形成“体验优先 + 稳定性优先”的 AI 产品开发意识，能够平衡模型效果、系统复杂度与交付效率。
